1. 了解Git和GitHub

1.1 Git是什么

Git是目前世界上最先进的分布式版本控制系统，结束手动管理多个“版本”的史前时代，进入到版本控制的20世纪，但是注意要提交到GitHub上才可以保存该版本

1.2 GitHub是什么

摘自与维基百科：GitHub（https://github.com/）是通过Git进行版本控制的软件源代码托管服务平台；GitHub是最流行的Git访问站点；个人和组织创建和访问保管中的代码；共同软件开发的功能，即社群功能；允许用户追踪其他用户、组织、软件库的动态，对软件代码的改动和bug提出评论；GitHub也提供了图表功能，用于概观显示开发者们怎样在代码库上工作以及软件的开发活跃程度

Github可以托管各种git库，并提供一个web界面（用户名.github.io/仓库名）

1.3 两者关系

Git是版本控制软件

Github是项目代码托管的平台，借助git来管理项目代码

1.4 为什么要学GitHub

  1. 实时跟踪和学习最新的行业内动态，关注行业内大牛，获取最新资料

  2. 便于托管自己的项目，方便查看各个版次

  3. 与别人合作的交流成本降低，便于组内交流

2. GitHub基本使用

2.1 注册、安装和配置

参考：https://help.github.com/cn/github/getting-started-with-github/using-github

2.2 GitHub中的基本概念

  1. 仓库（Repository）：即你的项目，你想在 GitHub 上新建项目，也就是新建一个 Repository，分为私有仓库和开源仓库；仓库用来存放项目代码，每个项目对应一个仓库，多个开源项目则有多个仓库；比如我用GitHub来存放我的blog文件的时候就新建了对应的仓库，新建仓库的同时，其内的配置文件也会生成不用额外的操作

  2. 收藏（Star）：收藏项目，方便下次查看；一般是你觉得很好的一个项目或者与个人研究相关的学习资料，收藏之后在自己的主页就可以很方便的调出来学习该项目的代码和内部托管的知识

  3. 复制克隆项目（Fork）：相当于项目的分叉，就是说你看到别人的一个开源项目，然后你想用到自己的研究中，你就可以Fork一份，这个项目也就复制到你的主页中，你可以对它进行修改，根据自己项目的需求来修改，不会对别人原有的的项目产生干扰，也就是fork的项目和原有的是独立存在的

  4. 发起请求（Pull Request）：基于 Fork，你在别人的基础上做了改进，后来觉得改进的很不错，应该要把这些改进让更多的人收益，于是就想把自己的改进合并到原有项目里，这个时候你就可以发起一个 Pull Request（简称PR） ，原有项目创建人就可以收到这个请求，这个时候他会仔细review你的代码，并且测试觉得OK了，就会接受你的PR，这个时候你做的改进原有项目也会拥有

  5. 关注（Watch）：关注项目，当项目更新可以接收到通知

  6. 事务卡片（Issue）：发现代码BUG，但是目前没有成型代码，需要讨论时用

2.3 基础使用指南

  1. 其实目前自己在本地写blog然后托管到GitHub上的行为也是一种基础使用

  2. 目前有两个GitHub账号，一个专门作为博客的托管使用，另一个作为优秀项目的学习

  3. 每个项目里面最重要的是redeme指南，作者会在里面写一些关于这个项目的想法和介绍，让你对该项目有个直观的理解，然后就是里面的代码和配套的图片也是很值得学习的，fork或者star一个项目之后对整个库先浏览一遍，然后找到关键内容学习

  4. 多关注一些感兴趣或者和自己研究密切相关的项目，可以设置提醒
